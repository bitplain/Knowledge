# Рекомендуемая структура проекта

Ниже приведён ориентир по структуре папок и файлов для масштабируемой базы знаний. Она учитывает контентные разделы, статичные ассеты и шаблоны, чтобы поддерживать единый стиль и упрощать обновления.

## Дерево каталогов

```text
.
├── mkdocs.yml                    # Конфигурация сайта с комментариями
└── docs/
    ├── index.md                  # Главная страница
    ├── guides/                   # Руководства и методологии
    │   ├── structure.md          # Структура проекта
    │   ├── configuration.md      # Настройки MkDocs
    │   ├── plugins.md            # Рекомендуемые плагины и расширения
    │   ├── content.md            # Организация контента и навигации
    │   └── material-examples.md  # Визуальные компоненты Material
    ├── knowledge/                # Основной контент по направлениям бизнеса
    │   ├── processes/            # Процессы и регламенты
    │   ├── engineering/          # Технические стандарты и гайды
    │   └── onboarding/           # Вводные материалы для сотрудников
    ├── playbooks/                # Пошаговые инструкции и рецепты
    ├── changelog/                # История изменений и релизные заметки
    ├── assets/
    │   ├── stylesheets/          # Кастомные стили
    │   ├── images/               # Иллюстрации и схемы
    │   └── attachments/          # Файлы для загрузки (pdf, xlsx и т.д.)
    └── overrides/                # Опциональные HTML-оверрайды для темы
```

### Назначение и преимущества

- **Разделение по доменам знаний** (`knowledge/`): облегчает навигацию и поиск, упрощает делегирование ответственности владельцам разделов.
- **Playbooks**: быстрый доступ к пошаговым процедурам, удобно для NOC/SRE и поддержки.
- **Changelog**: прозрачность изменений, информирование пользователей о нововведениях и обновлениях документации.
- **Assets и overrides**: единая точка для стилей, изображений и шаблонов, что сохраняет целостность визуальной части.

### Практические советы

- Используйте короткие и говорящие имена файлов (`incident-response.md`, `api-guidelines.md`).
- Храните изображения рядом с материалами, но в подпапке `images/`, чтобы избежать засорения корня раздела.
- Для крупных разделов добавляйте `README.md` с кратким описанием содержимого.
- Поддерживайте зеркальную структуру в навигации `nav:` в `mkdocs.yml`, чтобы путь в URL отражал логику каталогов.
