# Рекомендуемые плагины и расширения

Список проверенных плагинов и markdown-расширений, которые улучшают UX, навигацию и визуальную выразительность без усложнения поддержки.

## Плагины

| Плагин | Назначение | Конфигурация | Визуальный/функциональный эффект |
| --- | --- | --- | --- |
| `search` | Полнотекстовый поиск с подсветкой | `plugins: \n  - search: {lang: [ru, en]}` | Быстрый поиск по двум языкам, подсказки и подсветка совпадений. |
| `tags` | Тегирование страниц и авто-страница тегов | `plugins:\n  - tags` | Страницы получают блок тегов и автоматически строится индекс по тегам. |
| `git-revision-date-localized` | Вывод даты последнего изменения | `plugins:\n  - git-revision-date-localized: {type: datetime}` | Повышает доверие к актуальности материалов. |
| `glightbox` | Открытие изображений/видео в оверлее | `plugins:\n  - glightbox` | Плавный просмотр иллюстраций без ухода со страницы. |
| `section-index` | Поддержка `index.md` в подпапках | `plugins:\n  - section-index` | Кликабельные разделы в боковом меню, аккуратные URL вида `/knowledge/`. |

## Markdown-расширения

| Расширение | Назначение | Пример использования | Эффект |
| --- | --- | --- | --- |
| `admonition` + `pymdownx.details` | Выделение заметок и раскрывающихся блоков | `!!! note "Важно"` | Быстрая визуализация важных блоков, компактность через «спойлеры». |
| `pymdownx.tabbed` | Вкладки для вариантов инструкции | `=== "Linux"` | Один экран для нескольких платформ/ролей без дублирования текста. |
| `pymdownx.superfences` (mermaid) | Диаграммы последовательностей/блок-схемы | <code>```mermaid</code> | Наглядные схемы прямо в Markdown. |
| `pymdownx.highlight` + `inlinehilite` | Подсветка кода и строк | <code>```python hl_lines="2"</code> | Удобное чтение примеров кода, подсветка важных строк. |
| `pymdownx.emoji` | Поддержка SVG-эмодзи | `:rocket:` | Лёгкие акценты в текстах без тяжёлых изображений. |

### Почему именно они

- Все плагины открытые и поддерживаются сообществом, без Insider-функций.
- Минимум конфликтов: плагины отвечают за разные области (поиск, теги, даты, медиа, навигация).
- Низкая нагрузка: плагины не добавляют сложных сборок и хорошо работают на больших базах знаний.

### Советы по установке

```bash
pip install \
  mkdocs-material \
  mkdocs-glightbox \
  mkdocs-section-index \
  mkdocs-git-revision-date-localized-plugin
```

- Для тегов и поиска достаточно базового `mkdocs-material`.
- После установки прогоняйте `mkdocs build` — так вы поймёте, нет ли пропущенных зависимостей.
